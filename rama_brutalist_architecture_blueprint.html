The "RAMA Feel" — Interaction Logistics
Sticky → Horizontal Scroll Gallery

The gallery section is pinned via GSAP ScrollTrigger while the inner track translates on the X axis. The scroll distance consumed equals the total horizontal track width minus one viewport width — so the user "uses up" exactly the right amount of vertical scroll to traverse every card. Framer Motion's useScroll reads the pinned container's progress (0→1) and useTransform maps it to pixels.

// StickyGallery.tsx const containerRef = useRef(null); const { scrollYProgress } = useScroll({ target: containerRef, offset: ["start start", "end end"] }); // trackWidth = (cardWidth + gap) * numCards const x = useTransform( scrollYProgress, [0, 1], ["0px", `-${trackWidth - viewportW}px`] ); // Outer: height = trackWidth (consumes scroll budget) // Inner: position sticky, overflow hidden // Track: motion.div with x transform

The outer div height is set to trackWidth + 100vh — the extra viewport height gives the sticky panel time to settle before sliding begins. Each card also gets an individual useTransform with a slight parallax offset to create depth.

// CursorProvider.tsx (global, in layout.tsx) const [pos, setPos] = useState({ x: 0, y: 0 }); const [variant, setVariant] = useState("default"); // CSS: position fixed, pointer-events none // mix-blend-mode: difference ← KEY RULE // background: white → appears black on white bg // background: white → appears white on black bg // On interactive elements: <div onMouseEnter={() => setVariant("project")}> // cursor expands to 80px, shows "VIEW" label </div>

The cursor blob is always background: white. The difference blend mode inverts everything beneath it — so on a white section it becomes black, on a black hero it appears white. No JS color-detection needed.
